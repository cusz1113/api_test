# 接口数据格式定义（统一标准）

## 顶层结构

```json
{
  "meta": {
    "source_type": "openapi|swagger|yapi|postman|har|curl",
    "source_file": "原始文件名",
    "parsed_at": "ISO8601时间戳",
    "base_url": "基础URL（如有）",
    "total_endpoints": 61
  },
  "schemas": { ... },
  "endpoints": [ ... ]
}
```

### meta 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| source_type | ✅ | 来源类型：openapi / swagger / yapi / postman / har / curl |
| source_file | ✅ | 原始文件名或URL |
| parsed_at | ✅ | 解析时间 |
| base_url | ❌ | API基础URL，从文档中提取（如 servers[0].url） |
| total_endpoints | ✅ | 解析出的接口总数 |

---

## schemas — 数据模型定义

用于记录接口文档中定义的 Schema/Model，便于引用。

```json
{
  "schemas": {
    "UserInfoSchema": {
      "name": "UserInfoSchema",
      "type": "object",
      "description": "用户信息",
      "properties": {
        "id": {"type": "integer", "description": "用户ID"},
        "username": {"type": "string", "description": "用户名"},
        "email": {"type": "string", "format": "email", "description": "邮箱"}
      },
      "required": ["id", "username"]
    }
  }
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| name | ✅ | Schema名称 |
| type | ✅ | 类型：object / array / string / integer 等 |
| description | ❌ | 描述 |
| properties | ❌ | 对象属性（type=object时） |
| items | ❌ | 数组元素类型（type=array时） |
| required | ❌ | 必填字段列表 |
| enum | ❌ | 枚举值 |

---

## endpoints — 接口列表（核心）

```json
{
  "endpoints": [
    {
      "id": "user_login",
      "name": "用户登录",
      "description": "使用用户名和密码登录，返回token",
      "method": "POST",
      "path": "/api/users/login",
      "tags": ["用户管理"],
      "deprecated": false,
      "params": {
        "path": [],
        "query": [],
        "header": [],
        "cookie": []
      },
      "request_body": {
        "content_type": "application/json",
        "schema": {
          "type": "object",
          "properties": {
            "username": {"type": "string", "description": "用户名"},
            "password": {"type": "string", "description": "密码", "format": "password"}
          },
          "required": ["username", "password"]
        },
        "example": {
          "username": "admin",
          "password": "123456"
        }
      },
      "required_headers": [
        {"name": "Content-Type", "value": "application/json", "description": "请求体格式"}
      ],
      "responses": [
        {
          "status_code": 200,
          "description": "登录成功",
          "headers": {"Content-Type": "application/json"},
          "body": {
            "token": "eyJhbGciOiJIUzI1NiIs...",
            "token_type": "bearer"
          },
          "schema_ref": "TokenResponse"
        }
      ],
      "auth": {
        "type": "none",
        "description": "登录接口不需要认证"
      }
    }
  ]
}
```

### 接口基本信息

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| id | ✅ | string | 唯一标识，格式：`模块_操作`（如 user_login） |
| name | ✅ | string | 接口名称（优先取 summary，其次 operationId） |
| description | ✅ | string | 接口详细描述，无描述时为空字符串 |
| method | ✅ | string | HTTP方法：GET/POST/PUT/DELETE/PATCH/HEAD/OPTIONS |
| path | ✅ | string | 接口路径（含路径参数，如 /api/users/{id}） |
| tags | ✅ | string[] | 所属模块/分组标签 |
| deprecated | ❌ | boolean | 是否已废弃，默认 false |

### id 生成规则

格式：`{模块简写}_{操作}`，全部小写，下划线连接。

- 优先用 `operationId`（去掉特殊字符，转小写）
- 无 operationId 时：`{tag}_{method}_{path摘要}`
  - 例：`users_post_login`、`projects_get_list`
  - path 中的 `{id}` 去掉，用 path 最后一段
- 保证全局唯一，重复时追加序号：`user_login_2`

### params — 请求参数

```json
{
  "params": {
    "path": [
      {"name": "id", "type": "integer", "description": "用户ID", "required": true}
    ],
    "query": [
      {"name": "page", "type": "integer", "description": "页码", "required": false, "default": 1},
      {"name": "size", "type": "integer", "description": "每页数量", "required": false, "default": 10}
    ],
    "header": [
      {"name": "X-Request-Id", "type": "string", "description": "请求追踪ID", "required": false}
    ],
    "cookie": [
      {"name": "session_id", "type": "string", "description": "会话ID", "required": false}
    ]
  }
}
```

参数字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| name | ✅ | 参数名 |
| type | ✅ | 数据类型：string/integer/number/boolean/array/object |
| description | ❌ | 参数描述 |
| required | ❌ | 是否必填，默认 false |
| default | ❌ | 默认值 |
| enum | ❌ | 枚举值列表 |
| format | ❌ | 格式约束：email/mobile/date/date-time/uri/uuid/password 等 |
| example | ❌ | 示例值 |
| items | ❌ | 数组元素类型（type=array 时） |

### request_body — 请求体

**扁平化结构，去掉 schema 嵌套：**

```json
{
  "request_body": {
    "content_type": "application/json",
    "fields": [
      {
        "name": "username",
        "type": "string",
        "required": true,
        "description": "用户名",
        "maxLength": 20,
        "minLength": 6
      },
      {
        "name": "nickname",
        "type": "string",
        "required": false,
        "description": "昵称",
        "default": ""
      }
    ],
    "example": {
      "username": "testuser",
      "nickname": "测试用户"
    }
  }
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| content_type | ✅ | Content-Type：application/json / multipart/form-data / application/x-www-form-urlencoded 等 |
| fields | ✅ | 字段列表，每个字段包含 name/type/required/description/default/约束条件 |
| example | ✅ | 根据字段约束智能生成的示例值 |

**无请求体时**：`request_body` 设为 `null`

**多种 Content-Type 时**：只取第一个或最常用的（通常 application/json）

**fields 中的字段格式：**

| 字段 | 必填 | 说明 |
|------|------|------|
| name | ✅ | 字段名称 |
| type | ✅ | 数据类型：string/integer/number/boolean/array/object |
| required | ✅ | 是否必填 |
| description | ❌ | 字段描述 |
| default | ❌ | 默认值 |
| maxLength / minLength | ❌ | 长度约束 |
| format | ❌ | 格式约束：email/mobile/date/date-time/uri/uuid/password 等 |
| enum | ❌ | 枚举值 |
| pattern | ❌ | 正则约束 |
| minimum / maximum | ❌ | 数值范围 |
| schema_ref | ❌ | 原始 Schema 引用名称 |
| children | ❌ | 嵌套对象的子字段列表（type=object 时） |
| items_type | ❌ | 数组元素类型（type=array 时） |

**注意**：不再保留 `schema` 嵌套层和 `required` 数组，必填信息通过每个字段的 `required: true/false` 标识。

### required_headers — 必要请求头

```json
{
  "required_headers": [
    {"name": "Content-Type", "value": "application/json", "description": "请求体格式"}
  ]
}
```

自动推断规则：
- 有 request_body 且 content_type=json → 自动添加 Content-Type: application/json
- 有 request_body 且 content_type=form → 自动添加 Content-Type: multipart/form-data
- auth.type=bearer → 自动添加 Authorization: Bearer {token}
- auth.type=api_key 且指定了 header → 自动添加对应 header

### responses — 响应

**扁平化结构，与 request_body 一致：**

```json
{
  "responses": [
    {
      "status_code": 200,
      "description": "成功",
      "schema_ref": "UserInfoSchema",
      "fields": [
        {"name": "id", "type": "integer", "required": true, "description": "用户ID"},
        {"name": "username", "type": "string", "required": true, "description": "用户名"}
      ],
      "example": {"id": 1, "username": "testuser"}
    },
    {
      "status_code": 422,
      "description": "参数校验失败",
      "schema_ref": "HTTPValidationError",
      "fields": [
        {"name": "detail", "type": "array", "required": false, "description": "", "items_type": "object", "children": [...]}
      ],
      "example": {"detail": [{"loc": ["body", "username"], "msg": "field required", "type": "value_error.missing"}]}
    }
  ]
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| status_code | ✅ | 状态码 |
| description | ❌ | 状态描述 |
| schema_ref | ❌ | 引用的 Schema 名称 |
| fields | ✅ | 响应体字段列表（格式与 request_body.fields 一致） |
| example | ✅ | 根据字段智能生成的响应示例 |

提取规则：
- 优先取 200（或 201/204）的成功响应
- 同时取错误响应（400/401/403/404/422/500）
- 最多保留 3 个响应（1成功 + 2错误）
- **不再保留 `schema` 嵌套层**

### auth — 鉴权说明

```json
{
  "auth": {
    "type": "none|bearer|api_key|basic|oauth2|custom",
    "header_name": "Authorization",
    "description": "需要登录后访问"
  }
}
```

| type | 说明 | header_name |
|------|------|-------------|
| none | 无需认证 | - |
| bearer | Bearer Token | Authorization |
| api_key | API Key | 自定义（如 X-API-Key） |
| basic | HTTP Basic Auth | Authorization |
| oauth2 | OAuth2 | Authorization |
| custom | 自定义认证 | 自定义 |

推断规则（按优先级）：
1. 接口级 security 定义
2. 全局 security 定义
3. 请求头中的 Authorization / Cookie 中的 token
4. 无任何认证信息 → none

---

## 数据质量规则

1. **完整性**：文档中的每个接口都必须被解析，不得遗漏
2. **准确性**：参数类型、必填项、默认值必须和原文档一致
3. **示例优先**：有 example 时优先使用 example，没有时根据 schema 生成
4. **引用展开**：所有 $ref 必须展开为完整定义，方便直接使用
5. **去重**：同一接口不应出现多次（通过 path+method 去重）
6. **统计校验**：解析完成后，对比原文档的接口总数，确保一致
