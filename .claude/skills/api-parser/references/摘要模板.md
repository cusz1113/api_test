# 解析摘要模板

## 使用方式

解析完成后，用此模板生成 `api-spec-summary.md` 供用户审阅。

---

## 模板

```markdown
# 📄 接口文档解析摘要

## 基本信息

| 项目 | 值 |
|------|-----|
| 来源类型 | {source_type} |
| 源文件 | {source_file} |
| 解析时间 | {parsed_at} |
| 基础URL | {base_url 或 "未指定"} |
| 接口总数 | {total_endpoints} |
| 数据模型数 | {total_schemas} |
| 模块数 | {total_tags} |

## 模块分布

| 模块 | 接口数 | 需认证 | 公开 |
|------|--------|--------|------|
| {tag} | {count} | {auth_count} | {public_count} |

## 鉴权方式

| 认证类型 | 接口数 | 说明 |
|----------|--------|------|
| none | {count} | 无需认证 |
| bearer | {count} | Bearer Token |
| api_key | {count} | API Key |
| basic | {count} | Basic Auth |

## 接口列表

### {tag_name}

| 接口名称 | 方法 | 路径 | 鉴权 | 描述 |
|----------|------|------|------|------|
| {name} | {method} | {path} | {auth_type} | {description} |

---

✅ 解析完成，共 {total_endpoints} 个接口，{total_schemas} 个数据模型。
确认无误后可进入下一步。
```

## 注意事项

1. 模块按接口数降序排列
2. 接口列表中的描述截断到50字符
3. 如果接口数超过50，按模块分组只显示摘要，不逐个列出
4. 末尾附上统计信息供用户确认
