# 解析规则 — 各来源格式解析策略

## 通用规则

1. **解析前校验**：识别文件类型，校验格式有效性
2. **解析后校验**：对比原文档接口总数，确保无遗漏
3. **引用展开**：所有 Schema 引用（$ref）必须展开为完整定义
4. **路径规范化**：去除末尾 `/`（如 `/api/users/` → `/api/users`），保留路径参数 `{id}`
5. **方法大写**：统一转为 GET/POST/PUT/DELETE/PATCH
6. **标签归一化**：空标签归为 "未分类"

## 1. OpenAPI 3.x (openapi.json / openapi.yaml)

**识别标志**：顶层有 `"openapi": "3.x.x"`

**解析映射**：

| 目标字段 | OpenAPI 路径 |
|----------|-------------|
| name | `paths.{path}.{method}.summary` → `operationId` |
| description | `paths.{path}.{method}.description` |
| method | 路径的 HTTP method |
| path | paths 的 key |
| tags | `paths.{path}.{method}.tags` |
| deprecated | `paths.{path}.{method}.deprecated` |
| 路径参数 | `paths.{path}.{method}.parameters[location=path]` |
| 查询参数 | `paths.{path}.{method}.parameters[location=query]` |
| 请求头参数 | `paths.{path}.{method}.parameters[location=header]` |
| Cookie参数 | `paths.{path}.{method}.parameters[location=cookie]` |
| 请求体 | `paths.{path}.{method}.requestBody.content[mediaType]` |
| 必要请求头 | 从 content_type + security 推断 |
| 响应 | `paths.{path}.{method}.responses` |
| 鉴权 | `paths.{path}.{method}.security` → `components.securitySchemes` → 全局 `security` |
| 数据模型 | `components.schemas` |

**参数解析细节**：

```
parameters[].schema → type, format, enum, default
parameters[].description → description
parameters[].required → required
parameters[].example → example
parameters[].schema.items → items（数组类型时）
```

**请求体解析**：

```
requestBody.content["application/json"].schema → schema
requestBody.content["application/json"].example → example
requestBody.content["application/json"].schema.$ref → 展开引用
```

**鉴权解析**：

```
security → 遍历每个 scheme
components.securitySchemes[type=bearer] → auth.type = "bearer"
components.securitySchemes[type=apiKey + in=header] → auth.type = "api_key"
components.securitySchemes[type=http + scheme=basic] → auth.type = "basic"
```

## 2. Swagger 2.0 (swagger.json / swagger.yaml)

**识别标志**：顶层有 `"swagger": "2.0"`

**与 OpenAPI 3.x 的差异**：

| 差异点 | Swagger 2.0 | OpenAPI 3.x |
|--------|------------|-------------|
| body 参数 | 放在 `parameters[in=body]` 中 | `requestBody` |
| 文件上传 | `parameters[in=formData, type=file]` | `requestBody[multipart/form-data]` |
| 响应schema | `responses.{code}.schema` | `responses.{code}.content[media].schema` |
| 安全定义 | `securityDefinitions` | `components.securitySchemes` |
| 参数引用 | `$ref: "#/parameters/xxx"` | `components.parameters` |

**body 参数特殊处理**：

```python
# Swagger 2.0 的 body 参数 → 转为 request_body
for param in parameters:
    if param["in"] == "body":
        request_body = {
            "content_type": "application/json",
            "schema": resolve_ref(param["schema"]),
            "example": param.get("example")
        }
    elif param["in"] == "formData":
        # formData 参数 → multipart/form-data 请求体
        # 收集所有 formData 参数，转为 request_body.schema.properties
```

**鉴权解析**：

```
securityDefinitions → 同 OpenAPI components.securitySchemes
type=bearer → auth.type = "bearer"
type=apiKey → auth.type = "api_key"
type=basic → auth.type = "basic"
```

## 3. YApi 导出 JSON

**识别标志**：顶层有 `"catname"` 或 `"apiData"` 或 `[{name: "分类", list: [...]}]` 格式

**YApi 导出结构**：

```json
[
  {
    "name": "用户管理",
    "index": 0,
    "list": [
      {
        "path": "/api/users/login",
        "method": "POST",
        "title": "用户登录",
        "desc": "使用用户名密码登录",
        "req_headers": [
          {"name": "Content-Type", "value": "application/json"}
        ],
        "req_query": [
          {"name": "page", "required": "1", "desc": "页码"}
        ],
        "req_params": [
          {"name": "id", "desc": "用户ID"}
        ],
        "req_body_type": "json",
        "req_body_form": [],
        "req_body_other": "{\"username\":\"admin\",\"password\":\"123\"}",
        "res_body": "{\"code\":200,\"data\":{...}}",
        "status": "done",
        "tag": ["登录", "认证"]
      }
    ]
  }
]
```

**解析映射**：

| 目标字段 | YApi 路径 |
|----------|----------|
| name | `item.title` |
| description | `item.desc` |
| method | `item.method` |
| path | `item.path` |
| tags | `cat.name` + `item.tag` |
| 路径参数 | `item.req_params` |
| 查询参数 | `item.req_query` |
| 请求头参数 | `item.req_headers` |
| 请求体 | 根据 `req_body_type` 解析 |
| 必要请求头 | `item.req_headers` |
| 响应 | `item.res_body` |
| 鉴权 | 从 req_headers 中的 Authorization 推断 |

**请求体解析（按 req_body_type）**：

| req_body_type | 处理方式 |
|---------------|---------|
| "json" | 解析 `req_body_other` JSON 字符串 → schema |
| "form" | 解析 `req_body_form` 数组 → schema |
| "raw" | `req_body_other` 作为 raw text |
| "file" | multipart/form-data |
| "none" / 空 | request_body = null |

**参数类型推断**（YApi 参数通常无 type）：

```
required="1" → required=true, 其他 → false
数字类默认值（1, 10, 100）→ type=integer
"true"/"false" → type=boolean
含 @ 的字符串 → type=string, format=email
其他 → type=string
```

**鉴权推断**：

- 检查 `req_headers` 是否包含 `Authorization`
- 包含 `Bearer` → auth.type = "bearer"
- 包含 token → auth.type = "bearer"

## 4. Postman Collection v2.1

**识别标志**：顶层有 `"info"` 和 `"item"` 数组，`info.schema` 含 `postman`

**Postman 结构**：

```json
{
  "info": {"name": "My API", "schema": "https://schema.getpostman.com/json/collection/v2.1.0/collection.json"},
  "item": [
    {
      "name": "用户管理",
      "item": [
        {
          "name": "用户登录",
          "request": {
            "method": "POST",
            "header": [
              {"key": "Content-Type", "value": "application/json"}
            ],
            "url": {
              "raw": "http://example.com/api/users/login?page=1",
              "host": ["http://example.com"],
              "path": ["api", "users", "login"],
              "query": [{"key": "page", "value": "1"}],
              "variable": [{"key": "id", "value": "{{id}}"}]
            },
            "body": {
              "mode": "raw",
              "raw": "{\"username\":\"admin\",\"password\":\"123\"}",
              "options": {"raw": {"language": "json"}}
            }
          },
          "response": [
            {
              "name": "登录成功",
              "status": "OK",
              "code": 200,
              "header": [{"key": "Content-Type", "value": "application/json"}],
              "body": "{\"token\":\"abc\"}"
            }
          ]
        }
      ]
    }
  ]
}
```

**解析映射**：

| 目标字段 | Postman 路径 |
|----------|-------------|
| name | `item.name` |
| description | `item.request.description` 或 `item.description` |
| method | `item.request.method` |
| path | `item.request.url.path` 拼接为 `/api/users/login` |
| tags | 父级 folder.name |
| 路径参数 | `item.request.url.variable`（key 含 `:` 或 `{}` 的） |
| 查询参数 | `item.request.url.query` |
| 请求头参数 | `item.request.header` |
| Cookie参数 | Postman 不直接支持，设为 `[]` |
| 请求体 | 根据 `item.request.body.mode` 解析 |
| 必要请求头 | `item.request.header` |
| 响应 | `item.response` |
| 鉴权 | `item.request.auth` |

**URL 解析**：

```
url.path = ["api", "users", "login"] → "/api/users/login"
url.raw 包含 ? → 解析 query 参数
url.variable → 路径参数（key 如 ":id" 或 "{{id}}"）
```

**请求体解析（按 body.mode）**：

| mode | 处理方式 |
|------|---------|
| "raw" | 解析 raw JSON 字符串 → schema + example |
| "urlencoded" | urlencoded 数组 → schema |
| "formdata" | formdata 数组 → schema |
| "file" | multipart/form-data |
| "none" / 空 | request_body = null |

**鉴权解析**：

```
request.auth.type:
  "bearer" → auth.type = "bearer"
  "apikey" → auth.type = "api_key", header_name = request.auth.apikey.key
  "basic" → auth.type = "basic"
  "oauth2" → auth.type = "oauth2"
  无 auth → 从 header 中的 Authorization 推断
```

## 5. HAR 抓包数据

**识别标志**：顶层有 `"log"` 字段，内含 `"entries"` 数组

**HAR 结构**：

```json
{
  "log": {
    "version": "1.2",
    "entries": [
      {
        "request": {
          "method": "POST",
          "url": "http://example.com/api/users/login?page=1",
n          "httpVersion": "HTTP/1.1",
          "headers": [
            {"name": "Content-Type", "value": "application/json"},
            {"name": "Authorization", "value": "Bearer eyJ..."}
          ],
          "cookies": [
            {"name": "session_id", "value": "abc123"}
          ],
          "queryString": [
            {"name": "page", "value": "1"}
          ],
          "postData": {
            "mimeType": "application/json",
            "text": "{\"username\":\"admin\",\"password\":\"123\"}"
          }
        },
        "response": {
          "status": 200,
          "statusText": "OK",
          "headers": [
            {"name": "Content-Type", "value": "application/json"}
          ],
          "content": {
            "size": 256,
            "mimeType": "application/json",
            "text": "{\"token\":\"eyJ...\"}"
          }
        }
      }
    ]
  }
}
```

**解析映射**：

| 目标字段 | HAR 路径 |
|----------|---------|
| name | 从 url path 最后一段 + method 自动生成 |
| description | 无（HAR 不含描述），设为空字符串 |
| method | `entry.request.method` |
| path | 从 `entry.request.url` 提取 path 部分 |
| tags | 从 url path 自动分组（如 `/api/users/...` → `用户管理`） |
| 路径参数 | 从 path 中的 `{xxx}` 或 `:xxx` 提取 |
| 查询参数 | `entry.request.queryString` |
| 请求头参数 | `entry.request.headers` |
| Cookie参数 | `entry.request.cookies` |
| 请求体 | 根据 `entry.request.postData.mimeType` 解析 |
| 必要请求头 | `entry.request.headers` |
| 响应 | `entry.response` |
| 鉴权 | 从 headers 中的 Authorization 推断 |

**URL 解析**：

```
entry.request.url = "http://example.com/api/users/123?page=1"
→ path = "/api/users/123"
→ query = [{"name": "page", "value": "1"}]
→ base_url = "http://example.com"
```

**请求体解析（按 postData.mimeType）**：

| mimeType | 处理方式 |
|----------|---------|
| application/json | 解析 `postData.text` JSON → schema + example |
| application/x-www-form-urlencoded | 解析 `postData.params` 或按 `&` 拆分 text |
| multipart/form-data | 解析 `postData.params` |
| text/plain | request_body = null，仅记录 content_type |

**响应解析**：

```
entry.response.status → status_code
entry.response.statusText → description
entry.response.headers → headers
entry.response.content.text → body（尝试 JSON 解析）
entry.response.content.mimeType → content_type
```

**标签自动分组规则**：

HAR 无显式标签，按 URL path 自动分组：

```
/api/users/login     → ["用户管理"]
/api/users/123       → ["用户管理"]
/api/projects/       → ["项目管理"]
/api/pro/envs        → ["测试环境"]
/api/run/cases/1     → ["测试运行"]
```

规则：取 path 的第二段（`/api/{module}/...` 中的 `{module}`），映射为中文标签。无法映射时用原始路径段。

**重复请求去重**：

HAR 可能包含大量重复请求（刷新页面等），按 `method + path（去掉路径参数值）` 去重，保留第一条。

```
POST /api/users/123 和 POST /api/users/456 → 合并为 POST /api/users/{id}
GET /api/users?page=1 和 GET /api/users?page=2 → 合并为 GET /api/users
```

**鉴权推断**：

- 检查 request.headers 中的 Authorization
- `Bearer xxx` → auth.type = "bearer"
- `Basic xxx` → auth.type = "basic"
- `API-Key: xxx` → auth.type = "api_key"
- Cookie 中含 token/session → auth.type = "bearer"

## 6. cURL 命令

**识别标志**：文本以 `curl` 开头，包含 `-X`、`-H`、`-d`、`--data`、`--header` 等 curl 参数

**输入形式**：可以是单个 curl 命令字符串，或多行 curl 命令（每行一个）

**cURL 示例**：

```bash
curl -X POST 'http://example.com/api/users/login' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer eyJhbGciOi...' \
  -d '{"username":"admin","password":"123456"}'

curl 'http://example.com/api/users?page=1&size=10' \
  -H 'Cookie: session_id=abc123'

curl -X DELETE 'http://example.com/api/users/123' \
  -H 'Authorization: Bearer eyJhbGciOi...'
```

**解析映射**：

| 目标字段 | cURL 解析方式 |
|----------|-------------|
| name | 从 url path 最后一段 + method 自动生成 |
| description | 无，设为空字符串 |
| method | `-X` 参数，无 `-X` 时：有 `-d`/`--data` → POST，否则 → GET |
| path | URL 中的 path 部分 |
| tags | 从 path 自动分组（同 HAR 规则） |
| 路径参数 | path 中的 `{xxx}` 或 `:xxx` |
| 查询参数 | URL 中 `?` 后面的参数 |
| 请求头参数 | `-H` / `--header` 参数 |
| Cookie参数 | 从 `-H 'Cookie: ...'` 中提取 |
| 请求体 | `-d` / `--data` / `--data-raw` / `--data-binary` / `-F` |
| 必要请求头 | 从 `-H` 中提取 Content-Type、Authorization |
| 响应 | cURL 无响应数据，设为空数组 |
| 鉴权 | 从 `-H 'Authorization: ...'` 推断 |

**cURL 参数解析规则**：

| 参数 | 说明 |
|------|------|
| `-X METHOD` / `--request METHOD` | HTTP 方法 |
| `-H 'Key: Value'` / `--header 'Key: Value'` | 请求头（可多个） |
| `-d 'data'` / `--data 'data'` | 请求体（默认 POST） |
| `--data-raw 'data'` | 原始请求体（不解析 $变量） |
| `--data-binary 'data'` | 二进制请求体 |
| `-F 'key=value'` / `--form 'key=value'` | multipart/form-data |
| `-b 'cookie'` / `--cookie 'cookie'` | Cookie |
| `-u user:pass` / `--user user:pass` | Basic Auth |
| `'URL'`（无参数的位置参数） | 请求 URL |

以下参数忽略：`--compressed`、`-k`/`--insecure`、`-L`/`--location`、`-s`/`--silent`、`-o`/`--output`

**请求体解析（按参数类型）**：

| 参数 | content_type | 处理方式 |
|------|-------------|---------|
| `-d '{json}'` | application/json | 解析 JSON → schema + example |
| `--data-raw 'text'` | application/json | 同 `-d` |
| `--data-binary 'data'` | application/octet-stream | 不解析 schema，仅记录 |
| `-F 'key=value'` | multipart/form-data | 解析 form 字段 |
| `-F 'file=@/path'` | multipart/form-data | 记录文件上传 |
| 无 body 参数 | 无 | request_body = null |

**多命令批量解析**：

```
1. 按 curl 关键字拆分为多个命令
2. 每个 curl 命令独立解析为一个 endpoint
3. 合并去重（同 method + path）
4. 自动分组 tags
```

**鉴权推断**：

```
-H 'Authorization: Bearer xxx' → auth.type = "bearer"
-H 'Authorization: Basic xxx' → auth.type = "basic"
-H 'X-API-Key: xxx'         → auth.type = "api_key", header_name = "X-API-Key"
-u user:pass                → auth.type = "basic"
-H 'Cookie: token=xxx'     → auth.type = "bearer"（从 Cookie 中推断）
无认证信息                   → auth.type = "none"
```

**注意事项**：

- cURL 无响应数据，`responses` 始终为空数组
- cURL 的 URL 可能被单引号/双引号包裹，解析时需去除
- 反斜杠 `\` 续行符需要先合并为单行
- `-d` 参数中的 JSON 可能被单引号包裹，需正确提取

## 7. 未知格式兜底

当文件类型无法识别时：

1. 尝试按 JSON 解析，检查顶层字段特征
2. 检查是否为 YAML（含 `openapi:` 或 `swagger:` 关键字）
3. 检查是否为文本，以 `curl` 开头 → cURL 格式
4. 提示用户手动指定来源类型
5. 支持用户指定 `--source-type` 参数强制使用某种解析器

支持的 `--source-type` 值：`openapi` | `swagger` | `yapi` | `postman` | `har` | `curl`

## 引用展开规则

所有引用的 Schema 必须就地展开：

```
引用格式：
  OpenAPI 3.x: "$ref": "#/components/schemas/UserInfo"
  Swagger 2.0: "$ref": "#/definitions/UserInfo"

展开策略：
  1. 找到引用目标
  2. 深拷贝目标内容
  3. 递归展开内部嵌套引用
  4. 替换原 $ref 为展开后的内容
  5. 保留 schema_ref 字段记录原始引用名称
```

循环引用处理：设置最大递归深度 10，超过则标记为 `"$ref: SchemaName (循环引用)"`。

## 解析校验清单

解析完成后必须逐项验证：

- [ ] 接口总数与原文档一致
- [ ] 每个 endpoint 的 id 全局唯一
- [ ] method 全部大写
- [ ] path 无末尾斜杠
- [ ] 所有引用已展开
- [ ] auth 字段每个接口都有值
- [ ] 至少包含一个成功响应（2xx），HAR/cURL 除外
- [ ] tags 非空（空标签归为 "未分类"）
